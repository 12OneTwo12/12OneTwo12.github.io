  - 하나의 트랜잭션 내에 JPA와 MyBatis가 혼재되어 **HikariCP Deadlock 발생** 했습니다.
  - **DB Replication 복제지연 문제**로 일관성 이슈가 있었습니다. 
  - 또한, 비효율적인 쿼리와 여러 도메인 로직의 강한 결합으로 인해 **시스템 전반의 성능 저하**가 발생하고 있었습니다.
- **[해결 및 역할]**
  - **HikariCP Deadlock 해결**: Deadlock의 근본 원인을 파악하고, 기술 스택을 **JPA로 일원화**하는 방향으로 리팩토링하여 문제를 해결했습니다.
  - **DB Replication 지연 문제 해결**: **AbstractRoutingDataSource**와 **AOP**를 활용하여 DataSource를 동적으로 분리함으로써, 데이터 조회 시에는 Slave DB를, 변경 시에는 Master DB를 사용하도록 하여 Replication 지연으로 인한 데이터 불일치 문제를 해결했습니다.
  - **API 성능 개선**:
    - **Redis 기반 캐싱**을 설계하고 구현하여 외부 API 호출이 잦은 API의 응답 속도를 개선했습니다.
    - 불필요한 서브쿼리 제거, 인덱스 최적화 등 **SQL 튜닝**을 통해 데이터베이스 성능을 향상시켰습니다.
  - **아키텍처 개선**: **이벤트 기반 비동기 처리**를 제안하고 도입하여, 하나의 로직에 혼재되어 있던 여러 도메인(예: 대출 승인 시 심사결과 저장, 문자 발송, 관리자 알림)의 관심사를 분리하고 성능을 최적화했습니다.
- **[성과]**
  - **서버 안정성 및 신뢰도 향상**: HikariCP Deadlock 및 DB 복제 지연 문제를 해결하여 서버의 안정성과 사용자 데이터의 신뢰도를 크게 향상시켰습니다.
  - **API 응답 속도 대폭 개선**:
    - SQL 튜닝을 통해 주요 API의 성능을 **87.66%** 개선했습니다 (평균 10.295초 → 1.27초).
    - Redis 기반 캐싱을 통해 외부 API 의존 API의 응답 속도를 **74.69%** 개선했습니다 (평균 5.132초 → 1.299초).

- **[주요 기술]**: `Spring Boot 2.6.7` `Spring Security` `Java 11` `JPA` `Mybatis` `Thymeleaf` `Redis`

- **프로젝트 개요**: PHP 기반의 레거시 배치 서버를 Spring Batch로 리뉴얼하며 발생한 문제를 해결하고, 대용량 데이터 처리 성능을 최적화한 프로젝트입니다.
- **기간**: 2024.05 ~ 2025.03 (4~6명)
- **[문제]**
  - PHP와 Crontab 기반으로 구축된 레거시 배치 서버는 관리가 어려웠으며, 이를 Spring Batch로 이전하는 과정에서 여러 Job이 동시에 실행될 때 **Metadata Table에 Deadlock 문제 발생**했습니다. 
  - 또한, **Tasklet 방식으로 대량의 데이터를 처리할 때 심각한 성능 저하 및 데이터 정합성 문제 발생**했습니다.
- **[해결 및 역할]**
  - **Spring Batch Deadlock 문제 해결**: Job 동시 실행 시 발생하는 Deadlock의 원인을 파악하고, 여러 해결 방안(Job 동시 실행 금지, Spring Batch 버전 업 등)을 검토했습니다. 최종적으로 현실성과 안정성을 고려하여 Metadata Table 관련 트랜잭션의 **Isolation Level을 변경**하여 문제를 해결했습니다.
  - **대용량 데이터 처리 성능 최적화**: 기존 Tasklet 방식의 단점을 보완하기 위해, **Chunk 기반 처리**와 **Partitioning** 방식을 도입하여 대용량 데이터를 병렬로 처리하도록 리팩토링하고 성능을 최적화했습니다.
- **[성과]**
  - **안정적인 배치 환경 구축**: Deadlock 문제를 해결하여 여러 배치 작업이 동시에 안정적으로 실행될 수 있는 환경을 구축했습니다.
  - **데이터 정합성 99.9% 확보**: 성능 최적화를 통해 데이터 정합성 문제 발생 확률을 **99.9%** 감소시켰습니다 (최대 트랜잭션 처리 시간 22분 → 0.01초).
  - **처리 시간 64% 단축**: 배치 성능을 최적화하여 데이터 처리 시간을 **64%** 단축했습니다 (13.27분 → 4.77분).
  - **기술 공유 및 문서화**: 문제 해결 과정을 사내에 공유하고, 다른 팀에서도 활용할 수 있도록 관련 내용을 문서화했습니다.
  - **[주요 기술]**: `Spring Boot 2.7.3` `Spring Batch` `Java 11` `JPA` `MyBatis`
