현상 — "같은 쿼리인데 8배 느리다"
  • MSA 전환 중 프로시저 파라미터를 bigint → varchar로 변경했더니 56ms → 457ms (8배 느려짐)
  • 개별 쿼리는 1~2ms인데, 프로시저 안에서는 200ms+ → 335배 차이
3단계 가설 검증
가설 1: 인덱스 문제 ❌ 인덱스 정상 확인
가설 2: 서브쿼리 문제 ❌ 서브쿼리 제거해도 동일
가설 3: EXPLAIN 경고 발견 ✅ Collation 불일치
"Cannot use ref access on index due to type or collation conversion"
근본 원인
  • 테이블: utf8mb3 (3바이트) vs 프로시저 파라미터: utf8mb4 (4바이트)
  • Collation 불일치 → MySQL이 인덱스를 사용하지 못하고 Full Table Scan (45,453 rows)
  • PREPARE STATEMENT 검증: bigint 바인딩 0.6ms vs varchar 바인딩 202ms (335배 차이)
해결: 2단계 마이그레이션
  • 1단계 (긴급): utf8mb3 지역변수로 임시 해결 → 457ms → 42ms
  • 2단계 (근본): 전체 DB utf8mb4 통일 + 엔진 전환
성과

  • 121개 테이블 utf8mb4_general_ci 통일
  • 109개 테이블 MyISAM → InnoDB 전환 (인덱스 키 제한 1000B → 3072B)
